# 01 系统架构设计（V1.2 第1周初稿）

## 1. 架构目标
- 围绕“采集→治理→因子→评分→推荐→复盘”主链路，建立可运行的最小生产架构。
- 分层解耦：API 服务、调度服务、数据层、前端展示层。

## 2. 目录分层
- `backend/`：FastAPI、任务编排、因子与评分引擎。
- `frontend/`：Vue3 页面与图表展示。
- `scripts/`：采集、质检、部署、巡检脚本。
- `infra/`：systemd 与 nginx 配置。
- `docs/`：设计与交付文档。
- `data/`：本地数据落盘与中间产物。
- `logs/`：运行日志。

## 3. 后端模块规划
- `app/api/v1`：推荐、个股、任务、复盘接口。
- `app/db/models`：ODS/DWD/DM/SYS 数据模型。
- `app/services`：采集服务、因子服务、评分服务。
- `app/tasks`：APScheduler 作业定义与编排。
- `app/core`：配置、日志、异常、常量。
