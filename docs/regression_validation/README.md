# 阶段性回归验证目录

本目录用于集中存放每个开发阶段完成后的方案核对、验证步骤、预期结果和实际结果。

## 使用原则

1. 每完成一个阶段，必须先对照主方案和当前执行计划确认范围一致。
2. 每个阶段必须记录可重复执行的验证命令、预期结果和本次实际结果。
3. 数据库相关验证必须确认只操作 `stock_research_v12`，不得影响已有 `public` 数据。
4. 前后端变更必须至少通过后端测试、前端构建、API 冒烟检查和 linter 检查。
5. 若发现偏离方案设计，必须先修正实现或更新已确认的计划，再进入下一阶段。

## 文档清单

- `phase_1_3_validation.md`：阶段 1 到阶段 3 的核对与验证报告。
- `phase_4_validation.md`：阶段 4 Tushare 采集框架和数据质量检查验证报告。
- `phase_validation_template.md`：后续阶段统一回归验证模板。

## 关联运维文档

- `../operations/phase_1_3_operations.md`：阶段 1 到阶段 3 的启动、访问、健康检查和故障处理步骤。
